# 액터 개요

## 1. 학생 (Student)

### 기본 정보
- **액터명**: 학생 (Student)
- **설명**: 통합 학습 관리 시스템의 주요 사용자로, 게임화된 교육 환경에서 학습 활동에 참여하는 사용자
- **유형**: 주요 액터 (Primary Actor)

### 역할 및 책임
- 교사가 생성한 퀘스트를 수행하여 학습 목표 달성
- 가챠 시스템을 통해 캐릭터를 수집하고 게임 요소 활용
- 다른 학생들과의 배틀을 통해 경쟁적 학습 환경 참여
- 개인 학습 진도와 성취도를 스스로 관리 및 확인
- 교사가 생성한 레이드 이벤트에 참여하여 협력적 학습 수행

### 주요 특성
- **클래스 소속**: 특정 교사의 클래스에 속함 (클래스 코드를 통해 식별)
- **게임화 요소**: 코랄(화폐)과 탐사데이터를 통해 진행도 측정
- **개인화**: 개인별 학습 스타일과 진도에 맞춘 맞춤형 학습 경험
- **사회적 학습**: 다른 학생들과의 상호작용을 통한 학습 동기 부여

### 시스템과의 상호작용
- 웹 브라우저를 통해 시스템에 접근
- 개인 계정으로 로그인하여 개인화된 학습 환경 이용
- 실시간으로 퀘스트 수행 결과와 보상 획득
- 다른 학생들과의 실시간 배틀 및 순위 경쟁 참여

### 제약사항
- 교사가 생성한 클래스에만 참여 가능
- 교사가 설정한 퀘스트와 레이드 범위 내에서만 활동 가능
- 개인 정보 보호를 위한 적절한 권한 제한

## 2. 교사 (Teacher)

### 기본 정보
- **액터명**: 교사 (Teacher)
- **설명**: 통합 학습 관리 시스템의 관리자로, 학생들의 학습을 지도하고 관리하는 교육 전문가
- **유형**: 주요 액터 (Primary Actor)

### 역할 및 책임
- 클래스를 생성하고 학생들을 관리하여 학습 환경 구성
- 개별 및 그룹 퀘스트를 생성하여 학생들의 학습 목표 설정
- 학생들이 제출한 과제와 활동을 검토하고 승인/피드백 제공
- 레이드 이벤트를 통해 협력적 학습 환경 조성
- 학생들의 학습 진도와 성취도를 모니터링하고 분석

### 주요 특성
- **교육 전문성**: 교육 과정 설계 및 학습 평가 전문 지식 보유
- **관리 기능**: 여러 클래스와 학생들을 동시에 관리 가능
- **데이터 분석**: 학생들의 학습 데이터를 분석하여 교육 효과 측정
- **피드백 제공**: 학생 개별 및 그룹 단위로 맞춤형 피드백 제공

### 시스템과의 상호작용
- 웹 브라우저를 통해 관리자 인터페이스 접근
- 교사 전용 계정으로 시스템 관리 기능 이용
- 실시간으로 학생들의 활동과 진도를 모니터링
- 데이터 분석 도구를 통해 학습 효과 측정 및 개선

### 제약사항
- 교사 인증을 통한 계정 생성 및 관리
- 소속 학교 정보 확인을 통한 신원 검증
- 학생 개인정보 보호를 위한 적절한 접근 권한 관리

## 액터 간 관계

### 계층적 관계
- **교사 → 학생**: 교사가 학생들을 관리하고 지도하는 상하 관계
- **클래스 중심**: 교사가 생성한 클래스에 학생들이 속하는 구조적 관계

### 상호작용 패턴
- **직접적 상호작용**: 교사가 생성한 퀘스트에 학생이 참여
- **간접적 상호작용**: 학생들의 활동 데이터가 교사의 모니터링 대상이 됨
- **피드백 루프**: 학생 활동 → 교사 모니터링 → 피드백 제공 → 학습 개선

### 공통 목표
- **학습 효과 극대화**: 게임화를 통한 학습 동기 부여 및 참여도 향상
- **개인화된 교육**: 학생 개별 특성에 맞는 맞춤형 학습 환경 제공
- **협력적 학습**: 레이드와 그룹 퀘스트를 통한 협력 학습 문화 조성

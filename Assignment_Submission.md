# 통합 학습 관리 시스템 - 유스케이스 다이어그램 과제

## 과제 정보

- **주차**: 9주차

- **도구**: StarUML 사용
- **제출일**: 2025.11.02 (일) 오후 11:59

## 프로젝트 개요

### 시스템명
**통합 학습 관리 시스템 (Integrated Learning Management System)**

### 시스템 설명
학생과 교사를 위한 게임화된 교육 플랫폼으로, 퀘스트 수행, 가챠 시스템, 배틀 시스템, 레이드 이벤트 등을 통해 학습 동기를 부여하고 참여도를 향상시키는 웹 기반 학습 관리 시스템입니다.

### 주요 특징
- **게임화 요소**: 코랄(화폐), 탐사데이터, 캐릭터 수집, 칭호 시스템
- **사회적 학습**: 학생 간 배틀, 협력 레이드, 그룹 퀘스트
- **개인화**: 맞춤형 학습 진도 관리 및 피드백
- **실시간 모니터링**: 교사의 실시간 학생 진도 추적

## 작성된 문서 목록

1. **UseCaseDiagram.md** - 유스케이스 다이어그램 개요 및 StarUML 구현 가이드
2. **ActorOverview.md** - 액터 개요 (학생, 교사)
3. **UseCaseOverview.md** - 유스케이스 개요 (16개 유스케이스 상세 설명)
4. **StarUML_Instructions.md** - StarUML에서 다이어그램 구현을 위한 단계별 가이드

## StarUML 구현 방법

### 1. 프로젝트 생성
1. StarUML 실행
2. File → New Project 선택
3. Project Type: "Use Case Diagram" 선택

### 2. 시스템 경계 설정
1. Toolbox에서 "System" 선택
2. 다이어그램에 시스템 박스 그리기
3. 시스템명: "통합 학습 관리 시스템" 입력

### 3. 액터 생성
1. Toolbox에서 "Actor" 선택
2. 시스템 외부에 두 개의 액터 생성:
   - 학생 (Student) - 왼쪽
   - 교사 (Teacher) - 오른쪽

### 4. 유스케이스 생성
시스템 내부에 다음 유스케이스들을 생성:

#### 학생 관련 유스케이스 (시스템 내부 왼쪽)
- 학생 로그인
- 퀘스트 조회 및 수행
- 가챠 시스템 이용
- 배틀 시스템 이용
- 컬렉션 관리
- 프로필 관리
- 레이드 참여
- 대시보드 조회

#### 교사 관련 유스케이스 (시스템 내부 오른쪽)
- 교사 로그인
- 교사 회원가입
- 클래스 생성 및 관리
- 학생 관리
- 퀘스트 생성 및 승인
- 레이드 생성 및 관리
- 학생 진도 모니터링
- 교사 대시보드 조회

### 5. 관계 설정
1. Toolbox에서 "Association" 선택
2. 학생 액터와 학생 관련 유스케이스들을 연결
3. 교사 액터와 교사 관련 유스케이스들을 연결

### 6. 다이어그램 정리
1. 유스케이스들을 논리적으로 그룹화하여 배치
2. 연결선이 겹치지 않도록 조정
3. 다이어그램 제목과 설명 추가

## 주요 액터 요약

### 학생 (Student)
- **역할**: 시스템의 주요 사용자, 게임화된 학습 환경에서 학습 활동 참여
- **주요 기능**: 퀘스트 수행, 가챠 이용, 배틀 참여, 컬렉션 관리, 레이드 참여

### 교사 (Teacher)
- **역할**: 시스템 관리자, 학생들의 학습을 지도하고 관리하는 교육 전문가
- **주요 기능**: 클래스 관리, 퀘스트 생성/승인, 레이드 관리, 학생 진도 모니터링
